01 / AI удешевляет артефакты

Объём больше не сигнал.

Новый вопрос: пережил ли intent fan-out.

Раньше
Усилие -> код
AI-эра
Intent Atom
LLM
Intent -> правдоподобные артефакты
AI резко удешевил производство кода, тестов, документации и PR descriptions. Когда артефакт дешевеет, его количество перестаёт быть сигналом. Главный вопрос меняется: не сколько мы произвели, а сохранили ли мы смысл.

02 / Измеряйте пайплайн

Измеряйте пайплайн, не человека.

Больше активности может означать меньше уверенности.

PRs Lead time Deploy frequency
Intent Agent work Evidence Review
Роль человека scope acceptance evidence design
Классические метрики измеряют output. Агент может улучшить эти числа и одновременно ухудшить систему: расширить scope, сместить смысл, написать тесты под собственную реализацию. Если человек всё чаще улучшает пайплайн, измерять нужно пайплайн.

03 / Capability pipeline

Intent проходит gates до capability.

Outcome отдельно: хороший build всё ещё может быть неверной ставкой.

Intent Спецификация Реализация Evidence Internal acceptance
Outcome usage / business impact
Пайплайн: intent, specification, implementation, evidence, internal acceptance, outcome. Outcome вынесен отдельно: пайплайн может идеально сохранить смысл, а продуктовая гипотеза может оказаться неверной. Это разные виды отказа.

04 / Пять observable-величин

Пять величин живут на пайплайне.

Intent: повторить failed checkout payment без duplicate charge.

SIB смысл спецификации / масса реализации
Yield где задачи вытекают из пайплайна
Cost токены + review + rework + steering
Drift мы всё ещё решаем исходную задачу?
pre-SIB готовность до запуска агента
На одном примере видны пять величин: SIB как баланс смысла и реализации, Yield как утечки, Cost как стоимость принятой capability, Drift как смысловой сдвиг и pre-SIB как готовность пути до запуска агента. Для всех действует принцип: unknown не good и не bad.

05 / Сигнал, не score

SIB — телескоп, а не scoreboard.

Pipeline Integrity Index — нормативный слой. SIB остаётся decomposable.

decomposable unknown виден не ранжировать людей
SIB не должен превращаться в общий индекс здоровья и KPI. Это дескриптивный сигнал: посмотри сюда, здесь дисбаланс. Нормативный управляющий слой, если он нужен, должен быть отдельным Pipeline Integrity Index с guardrails и без ранжирования людей.

06 / Evidence доказывает intent

Coverage не доказывает смысл.

Разомкнуть петлю самоподтверждения агента.

Intent Atom нет duplicate charge
Evidence Claim тест + сценарий + сигнал
Judge Receipt подтверждено / частично / слабо
deterministic checks mutation tests human sampling drift judge
Coverage отвечает на вопрос о строках, а нам нужно доказательство intent. Агент может написать реализацию, тесты под неё и объяснение, почему всё хорошо. Мы размыкаем петлю через Intent Atom, Evidence Claim и Judge Receipt с детерминированными якорями и мониторингом drift judge.

07 / Что нужно доказать

Это гипотезы, не законы.

H1 pre-SIB предсказывает internal acceptance лучше, чем PR count.
H2 Высокий output при низкой evidence confidence предсказывает rework.
H3 Semantic evidence confidence предсказывает регрессии лучше coverage.
Это не закон природы, а набор гипотез. pre-SIB должен предсказывать acceptance лучше, чем PR count; output при низкой evidence confidence должен предсказывать rework; semantic confidence должен предсказывать регрессии лучше coverage. Сначала валидирую на своём workflow.

08 / Тезис -> демо

AI удешевляет артефакты. SIB следит за смыслом.

код тесты docs PR смысл?
Дальше демо SpecGraph capability pipeline in vivo
Сжатый тезис: AI удешевляет артефакты. SIB наблюдает, сохраняет ли реализация смысл спецификации. SpecGraph превращает это наблюдение в управляемый capability pipeline. Дальше переход к демо.

Backup / Стабильность генерации

У генерации есть кривая трения.

PFI считает турбулентность. CSI измеряет стабилизацию.

нестабильные попыткиstable state
Backup для вопроса про стабильность генерации. Probabilistic Friction Index считает хаотичные итерации до стабильности, а Checkpoint Stability Index показывает, насколько уточнения вызывают лавинообразные изменения.
