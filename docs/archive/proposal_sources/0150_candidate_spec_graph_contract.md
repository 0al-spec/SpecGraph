# Source: Candidate Spec Graph Contract

Operator intent: after the event-storming intake exists, add the next typed
artifact in the autonomous idea-to-spec loop: a complete candidate specification
graph that remains pre-canonical until validation, pre-SIB/coherence metrics,
repair loops, and repository governance approve promotion.

The slice should not run prompt agents or generate specs from scratch. It should
normalize and validate structured candidate graph seed data against the
event-storming intake boundary so the next slice can score it with pre-SIB
metrics.
