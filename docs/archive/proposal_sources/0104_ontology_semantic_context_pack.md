# Ontology Semantic Context Pack

## Source Intent

The operator wants the Ontology-SpecGraph-SpecSpace line to produce useful
forward implementation work, not only more proposal text. Proposal `0100`
defined Ontology-grounded semantic control, and proposal `0103` added the first
machine-readable policy plus deterministic smoke classification.

The next bounded slice should make the accepted vocabulary, aliases, relations,
gaps, and governance evidence portable as a typed context artifact for later
supervisor and SpecSpace consumers.

## Requested Work

- Build a deterministic `ontology_semantic_context_pack` artifact from the
  existing `0060` import surfaces and `0103` semantic policy.
- Include accepted ontology terms, accepted relations, aliases, deprecated
  terms, relation conflicts, unresolved gaps, package metadata, and governance
  evidence.
- Preserve the review-only authority boundary: the context pack can guide prompt
  inputs and review surfaces, but cannot execute agents, mutate canonical specs,
  update ontology packages, or expand accepted terms without review.
- Keep the output local under `runs/` and generated by `make ontology-imports`.

## Follow-Up Shape

After this slice, the next useful bounded steps are the full semantic lint
report, then ontology delta candidates/review packets, then the SpecSpace and
supervisor invocation boundary that consumes these artifacts.
