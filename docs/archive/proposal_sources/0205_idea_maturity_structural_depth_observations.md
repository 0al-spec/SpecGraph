# 0205 Idea Maturity Structural Depth Observations

Source draft for proposal `0205`.

Idea Maturity should absorb the reusable structural observations from Product
Demo Depth Baseline so consumers do not need a separate demo-specific report to
answer basic questions such as:

- did the intake identify actors;
- did it identify commands and domain events;
- does the candidate have workflow topology;
- do candidate nodes carry requirements and acceptance criteria.

These observations remain counts, not a score. They should be emitted under
`groups.candidate_structure_depth` in the Metrics-owned Idea Maturity report.
