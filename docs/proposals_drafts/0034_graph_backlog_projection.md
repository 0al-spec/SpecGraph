# Graph Backlog Projection Draft

SpecGraph should stop treating `tasks.md` as the primary work-discovery
surface. Existing derived artifacts already know which gaps are open, but they
are scattered across graph health, proposal runtime, trace, evidence, external
consumer, SpecPM, Metrics, and metric-threshold reports.

The next viewer-facing layer should normalize those gaps into one derived
backlog projection with source artifact links, priority, domain, subject id,
and next gap. The projection remains advisory and must not mutate canonical
specs or approve downstream feedback automatically.
