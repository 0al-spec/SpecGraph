# Ontology Semantic Context Pack

RFC: SG-RFC-0104
Version: 0.1.0

## Status

Implemented

Decision scope: deterministic context artifact that packages imported ontology
terms, relations, aliases, gaps, and governance evidence for later supervisor
and SpecSpace consumers.

This document does not build the full semantic lint report, run prompt agents,
generate ontology deltas, mutate canonical specs, create an ontology lockfile,
write ontology packages, or build SpecSpace UI.

## Summary

SpecGraph now emits a typed ontology semantic context pack:

```text
runs/ontology_semantic_context_pack.json
```

The pack is generated by `make ontology-imports` from:

- `ontology_package_index`;
- `ontology_binding_preview`;
- `ontology_import_gap_index`;
- `ontology_governance_evidence_index`;
- `tools/ontology_semantic_control_policy.json`.

The context pack gives later supervisor or SpecSpace consumers a bounded input
artifact with accepted terms, accepted relations, aliases, deprecated terms,
relation conflicts, unresolved ontology gaps, package metadata, and governance
evidence. It is not canonical authority and cannot mutate specs or Ontology
packages.

## Goals

- Add `ontology_semantic_context_pack` to the semantic control policy layout.
- Define a `semantic_context_pack_contract` with target scope and consumer
  boundary.
- Build the context pack deterministically in `tools/ontology_imports.py`.
- Emit the context pack from `make ontology-imports`.
- Keep the artifact under `runs/` and review-only.
- Cover artifact shape, counts, write path, and authority boundary in tests.

## Non-Goals

- Building `runs/ontology_semantic_lint_report.json`.
- Parsing arbitrary natural language.
- Invoking LLM prompt agents.
- Generating ontology delta candidates or review packets.
- Invoking `ontologyc`.
- Writing ontology packages or ontology lockfiles.
- Mutating `specs/nodes/*.yaml`.
- Adding SpecSpace UI, mutation routes, or review action APIs.

## Runtime Contract

The context pack declares:

```json
{
  "artifact_kind": "ontology_semantic_context_pack",
  "schema_version": 1,
  "proposal_id": "0104",
  "canonical_mutations_allowed": false,
  "tracked_artifacts_written": false,
  "target_scope": {
    "target_kind": "proposal",
    "target_ref": "SG-RFC-0104"
  },
  "packages": [],
  "accepted_terms": [],
  "accepted_relations": [],
  "aliases": [],
  "deprecated_terms": [],
  "relation_conflicts": [],
  "unresolved_gaps": [],
  "governance_evidence": []
}
```

The checked-in fixture currently produces:

- one imported ontology package;
- one accepted class term: `examcalc:Exam`;
- one accepted relation: `examcalc:requires_policy`;
- one accepted alias: `requires policy`;
- one deprecated term: `ExamPolicy`;
- one relation conflict: `allows policy`;
- one unresolved ontology gap: `examcalc:CASFunction`;
- one governance evidence item.

## Consumer Boundary

The context pack may be used as prompt-agent input context and as a SpecSpace
review surface source.

The context pack may not:

- execute prompt agents;
- expand accepted terms without review;
- mutate canonical specs;
- write ontology packages;
- update an import lockfile;
- become authority for lint reports or ontology deltas.

## Acceptance
